# Contributing

## Workflow

- Create a feature branch from `main`.
- Keep pull requests focused and small when possible.
- Add or update tests for behavior changes.
- Ensure local linting and test commands pass before opening a PR.

## Commit Guidance

- Use clear imperative commit messages, for example: `add backend health endpoint`.
- Group related changes in a single commit.

## Pull Request Checklist

- [ ] Code is formatted and linted
- [ ] Tests added or updated where applicable
- [ ] Local test suite passes
- [ ] Documentation updated for behavior or setup changes

## Local Commands

- `./scripts/setup.sh`
- `./scripts/lint.sh`
- `./scripts/test.sh`
- `./scripts/format.sh`
